# OPC Profile Memory Map

Status: OPTIONAL / PROFILE MEMORY MAP
Scope: `wiki/freelancer/opc/`

## Purpose

This file summarizes the intended memory scope for each Hermes Agent OPC profile in this workspace.

The map is organizational only. It does not grant runtime authority, Runes Shield authority, or automatic trusted-memory write authority.

## Core Rule

```text
secretary owns user-facing consent
runes-holder owns Runes Shield access
Hermes Runes MD Wiki remains Shield-governed
```

## Profiles

| Profile | Memory scope | Boundary |
|---|---|---|
| secretary | User-facing preferences, consent habits, reply style, Lark-facing conventions. | Does not own technical truth or repo verification. |
| coordinator | Task routing, profile selection, handoff discipline, one-profile-at-a-time constraints. | Does not own canonical project conclusions. |
| researcher | Source validation, freshness policy, citation policy, evidence comparison. | Does not convert unverified speculation into trusted memory. |
| writer | Documentation style, Traditional Chinese phrasing, report/presentation conventions, source-status wording. | Does not treat draft prose as source-of-truth. |
| builder | Repo operations, verification commands, smoke tests, deployment notes, local limitations. | Does not store sensitive runtime secrets or unverified commands as stable procedures. |
| runes-holder | Runes Shield usage, policy read order, proposal status, trusted/draft/rejected boundaries, placement guidance. | Does not own user consent or promotion authority. |

## Access Reminder

Other profiles may need Runes context, but actual Runes operations should be delegated to `runes-holder` and remain governed by Runes Shield.
