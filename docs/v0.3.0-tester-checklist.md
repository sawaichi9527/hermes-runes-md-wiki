# v0.3.0 Tester Checklist

Status: public tester checklist / pre-tag runtime surface review
Date: 2026-06-08
Baseline: `v0.3.0`

## Purpose

Use this checklist to confirm a fresh public tester checkout is using the intended `v0.3.0` runtime surface.

This checklist is intentionally short and runtime-facing. Developer history and milestone evidence live under `dev/` and are not required for normal tester onboarding.

## Expected Version

```bash
cat VERSION
```

Expected:

```text
0.3.0
```

`v0.1.0-beta.1` is superseded and should not be used for fresh tester onboarding.

## Runtime Wiki Seed

Expected runtime wiki surface:

```text
wiki/
  README.md
  hermes_runes_index.md
  _system/
  freelancer/
```

`freelancer` is the current dogfood workspace slug. Other testers should use their own lowercase hostname as the workspace slug.

## Runtime System Policy Surface

`wiki/_system/` should contain active runtime policy and guidance only.

It should not contain milestone or historical development files such as:

```text
m34-*.md
m35-*.md
m36-*.md
m37-*.md
m58-*.md
m62-*.md
p0_*.md
```

Those files belong under:

```text
dev/wiki-history/_system/
```

## Forge Inbox Boundary

Each workspace may contain:

```text
wiki/<workspace-slug>/forge-inbox/
```

`forge-inbox/` is a local governed draft inbox.

It is for unreviewed proposed memory changes before human review.

Files in `forge-inbox/` are not trusted memory.

Public committed content should normally be limited to:

```text
.gitkeep
README.md
```

Real local proposal drafts are working artifacts and should normally remain uncommitted.

## Default Bootstrap Dependency Profile

Default setup:

```bash
bash ./bin/hermes-memory-bootstrap
```

Expected behavior:

- installs core requirements only
- skips embedding requirements by default
- does not install torch, CUDA, sentence-transformers, transformers, or Triton by default

Optional embedding setup is explicit:

```bash
bash ./bin/hermes-memory-bootstrap --with-embedding
```

## Developer History Boundary

Normal testers should not import or edit `dev/` for routine runtime use.

`dev/` contains:

- milestone history
- old verification evidence
- developer-only fixtures
- release planning notes
- historical wiki material

Runtime user memory belongs under:

```text
wiki/<workspace-slug>/
```

System policy belongs under:

```text
wiki/_system/
```

## Safety Boundary

Do not put real secrets in Markdown memory, including:

- passwords
- API keys
- bot tokens
- database credentials
- private keys
- raw sensitive logs

## Quick Local Sanity Checks

```bash
cat VERSION

find wiki/_system -maxdepth 1 -type f \
  \( -name 'm[0-9]*.md' -o -name 'p0_*.md' \) \
  -printf '%f\n' | sort

find wiki/freelancer/forge-inbox -maxdepth 1 -type f -printf '%f\n' | sort

bash ./bin/hermes-memory-bootstrap
```

Expected:

```text
VERSION: 0.3.0
runtime milestone/p0 docs: no output
forge-inbox committed docs: .gitkeep and README.md
bootstrap: PASS / core requirements installed / embedding requirements skipped
```
