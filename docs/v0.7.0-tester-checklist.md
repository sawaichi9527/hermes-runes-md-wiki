## Governance boundary

The v0.7.0 fresh-install path must preserve:

- no real secrets in Markdown or git
- no direct trusted wiki mutation by external agents
- forge inbox remains draft / untrusted proposal space
- `forge-inbox/` content is not imported as trusted memory
- PostgreSQL credentials stay in local `.env`
- generated local runtime files remain uncommitted

## Reset validation

Strict reinstall testing should keep PostgreSQL stack/data removal separate from repository and venv removal.

Use the relevant sections in:

```text
docs/fresh-install-manual.md
```

Do not combine database deletion with repo deletion unless explicitly performing a full strict clean reinstall.
