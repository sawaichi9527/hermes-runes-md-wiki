# v0.5.0 Release Readiness

Status: final-ready  
Version line: 0.5.0  
Date: 2026-06-08

## Purpose

v0.5.0 is the post-v0.3.0 runtime-readiness line for Hermes Runes MD Wiki.

It adds CPU embedding enablement, restored hybrid/vector recall, fresh runtime smoke fixture alignment, accepted smoke gates for unavailable local fixtures/endpoints, and runtime surface cleanup.

## Major changes since v0.3.0

- CPU-only embedding writer enabled.
- PostgreSQL / pgvector backend verified with embeddings filled.
- Hybrid/vector recall restored for the current runtime workspace.
- Full smoke fixture alignment updated for the current `freelancer` runtime seed.
- LLM endpoint unavailable condition gated as accepted SKIP.
- Promotion/retrieval governance fixture absence gated as accepted SKIP for fresh runtime workspaces.
- Runtime CLI / tools surface cleaned:
  - runtime wrappers remain under `bin/`
  - runtime/support tools remain under root `tools/`
  - development-only smoke, fixtures, regression, trial, and historical assets moved under `dev/`
- Runtime support tool filenames no longer carry old milestone suffixes.
- `security_scan.py` resolves the current repository root instead of old `~/workspace/hermes-memory`.

## Current accepted SKIP gates

- `TB-M1989-FS001`: LLM endpoint unavailable for answer-generation smoke.
- `TB-M1989-FS002`: promotion governance fixture not present in fresh runtime workspace.
- `TB-M19810-RG001`: retrieval governance fixture not present in fresh runtime workspace.

These are accepted for v0.5.0 release-readiness as long as the main runtime smoke, backend, migration, security scan, and hybrid recall pass.

## Release gate commands

Run these commands from the repository root:

```bash
git status --short
cat VERSION
```

Run Runes policy checks:

```bash
./bin/hermes-runes decipher freshness --json
./bin/hermes-runes probe policy --json
```

Run backend and migration checks:

```bash
./bin/hermes-backend-check
./bin/hermes-memory-migrate
./bin/hermes-memory-security-scan
```

Run hybrid recall check:

```bash
./bin/hermes-recall "forge inbox boundary" \
  --project freelancer \
  --mode hybrid \
  --limit 5 \
  --json
```

Run smoke checks:

```bash
./bin/hermes-memory-smoke
./bin/hermes-retrieval-governance-smoke
```
